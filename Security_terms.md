**AIM (Access and Identity Management)** — управление личностями и доступом пользователей: кто ты, как ты входишь (аутентификация) и что тебе разрешено (авторизация), плюс лог кто-когда-куда заходил.

**RBAC (Role-Based Access Control)** — доступ выдаётся через роли (админ, оператор, зритель), а не персонально каждому.

**ABAC (Attribute-Based Access Control)** — доступ определяется атрибутами: отдел, локация, время, устройство — гибче, чем RBAC.

**Least Privilege** — каждому пользователю/процессу даётся минимум прав, необходимых для работы, не больше.

**Separation of Duties** — критичные операции разделены между разными людьми, чтобы один человек не мог и инициировать, и подтвердить действие сам.

**PAM (Privileged Access Management)** — централизованный контроль и аудит административного доступа (серверы, сетевое оборудование).

**TOTP (Time-based One-Time Password)** — временный код, меняющийся каждые N секунд, для 2FA.

**Push-аутентификация** — подтверждение входа через уведомление в приложении на телефоне.

**U2F / FIDO2** — аппаратные ключи (например, YubiKey) для сильной двухфакторной аутентификации без пароля.

**Active Directory (AD)** — каталог-сервис Windows, хранит пользователей, компьютеры и ресурсы домена для централизованного управления.

**GPO (Group Policy Objects)** — политики, которые администратор применяет ко всем машинам домена: пароли, firewall, разрешённые программы и т.д.

**TCP (Transmission Control Protocol)** — надёжный протокол с подтверждением доставки и контролем порядка пакетов; для веба, почты.

**UDP (User Datagram Protocol)** — протокол без гарантии доставки, зато быстрый; для видео и игр в реальном времени.

**Firewall** — фильтрует сетевой трафик по заданным правилам.

**Malware** — вирусы, черви, трояны, ransomware (шифрует данные и требует выкуп, пример: LockBit).

**RPO (Recovery Point Objective)** — сколько данных допустимо потерять при сбое (интервал между бэкапами).

**RTO (Recovery Time Objective)** — за сколько времени система должна восстановиться после сбоя.

**Шифрование дисков** — BitLocker / EFS (Windows), LUKS (Linux) — защищает данные при физической краже устройства.

**GPG** — шифрование отдельных файлов и сообщений.

**HackTheBox** — платформа для практики взлома в безопасной среде (поиск уязвимостей, privilege escalation, lateral movement).

**Lateral movement** — перемещение атакующего с одной скомпрометированной машины на другие внутри сети.

**Wireshark** — анализатор сетевого трафика на уровне пакетов.

**REST vs GraphQL** — REST: ресурсы + HTTP-методы (GET/POST/PUT/DELETE). GraphQL: клиент сам описывает, какие данные нужны, меньше лишнего трафика.

**XSS (Cross-Site Scripting)** — внедрение вредоносного JS в страницу, исполняется в браузере жертвы; защита — валидация ввода и правильный рендеринг.

**CSRF (Cross-Site Request Forgery)** — заставить браузер жертвы выполнить действие без её ведома; защита — CSRF-токены.

**SQL Injection** — внедрение SQL-кода через ввод пользователя; защита — параметризованные запросы.

**Redux** — управление состоянием во фронтенд-приложении.

**TypeScript vs JavaScript** — TS добавляет статическую типизацию, ошибки ловятся на этапе компиляции, код становится структурированнее.

**Tree Shaking** — удаление неиспользуемого кода из финального бандла.

**Lazy loading / оптимизация изображений** — ускоряют загрузку страницы, подгружая контент по мере надобности.

**Виртуализация списков (list virtualization)** — рендерятся только видимые элементы длинного списка, а не все тысячи сразу.

**JSON vs бинарные форматы** — бинарные форматы компактнее и быстрее передаются, чем JSON.

**Cypress (E2E)** — эмулирует реальные действия пользователя в браузере (клики, ввод, навигация), проверяя всю систему от фронта до бэка.

**Виртуализация инфраструктуры** — VM, контейнеры, облака — изоляция сред, масштабирование ресурсов, быстрое восстановление после сбоев.
